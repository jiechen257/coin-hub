# 系统中文化与操作手册设计

## 目标

将 Coin Hub 当前面向用户的 Web 控制台统一为中文界面，并补充一份独立的操作手册，降低首次使用和日常操作成本。

## 范围

- 将登录页、顶部壳层、配置页、命令中心、分析页、回放页、运行历史页的用户可见文案改为中文。
- 将根布局语言标记调整为 `zh-CN`，同步更新站点描述文案。
- 保持品牌名 `Coin Hub` 不变，只调整功能性文案和说明文案。
- 新增一份面向使用者的 Markdown 操作手册，默认放在 `doc/` 目录。

## 非目标

- 不引入完整国际化框架。
- 不增加中英文切换能力。
- 不调整接口协议、数据结构和业务流程。

## 设计决策

### 1. 采用单语言直接替换，而不是引入 i18n

当前需求是“系统使用中文”，而不是“支持多语言”。直接替换现有英文文案能在最小改动下完成目标，避免把一次文案统一扩成框架级重构。

### 2. 品牌名保留英文

`Coin Hub` 作为产品名继续保留，避免出现“品牌名翻译”和“功能文案翻译”混在一起的问题。页面标题、按钮、标签、提示语统一为中文。

### 3. 文档职责分离

- `README.md` 继续承担开发环境启动和工程脚本说明。
- 新增操作手册承担产品使用说明，包括登录、配置版本管理、命令中心、分析、回放和运行历史。

## 风险与处理

- 风险：界面中可能存在零散英文提示、状态文案或演示数据。
  处理：同时检查页面标题、组件标签、提示信息、空态、成功/失败消息以及分析页示例数据。

- 风险：测试仍使用英文选择器导致回归。
  处理：同步更新单元测试和 E2E 断言。

## 验证策略

- 运行证据：执行相关单元测试、E2E 测试和构建。
- 行为验证：确认关键页面标题、表单标签、按钮文案、操作反馈和根布局 `lang="zh-CN"` 已生效。
