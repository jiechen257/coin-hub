# Coin Hub 操作手册

## 1. 系统简介

Coin Hub 是一个单用户策略控制台，用于统一完成以下工作：

- 登录并进入受保护控制台
- 管理策略配置版本
- 在命令中心查看 BTC / ETH 最新分析结果
- 在分析页查看图表、信号、证据和观点
- 提交回放任务并查看结果
- 查看历史运行快照

## 2. 使用前提

使用系统前，请确认以下服务已经启动：

- Web 服务：`pnpm dev`
- Worker：`pnpm worker`

如果是首次启动，还需要先完成：

1. `pnpm install`
2. `pnpm prisma migrate dev`
3. `pnpm tsx prisma/seed.ts`

默认登录密码来自 `.env` 中的 `APP_PASSWORD`。

说明：当前版本不需要单独启动 `scheduler`。任务轮询已经内置在 `pnpm worker` 中。

## 3. 登录系统

1. 打开登录页 `/login`
2. 在“登录密码”输入框中输入密码
3. 点击“登录”
4. 登录成功后系统会自动跳转到“配置版本管理”页面

如果密码错误，页面会提示“密码不正确，请重试”。

## 4. 页面说明

### 4.1 配置版本管理

用途：创建新的策略配置版本，并切换当前生效版本。

页面分为两部分：

- 左侧：当前生效版本信息和新版本创建表单
- 右侧：历史版本列表

创建新版本步骤：

1. 在“摘要”中填写本次配置变更说明
2. 在“风险比例”中填写数值
3. 点击“创建新版本”
4. 页面刷新后，新版本会自动成为“生效中”

切换历史版本步骤：

1. 在“版本历史”中找到目标版本
2. 点击“设为生效”
3. 页面刷新后，目标版本会显示“生效中”

### 4.2 命令中心

用途：查看最新双资产分析快照，并手动触发一次新的分析任务。

页面内容包括：

- BTC / ETH 资产信号卡片
- 运行风险面板
- “运行分析”按钮

手动触发分析步骤：

1. 打开“命令中心”
2. 点击“运行分析”
3. 等待页面提示“分析任务已入队”
4. 系统会自动刷新最新快照

### 4.3 分析工作台

用途：从图表视角查看某个资产在不同周期下的分析结果。

可执行操作：

- 在 BTC / ETH 之间切换
- 在 `15m`、`1h`、`4h`、`1d` 间切换
- 查看当前信号、告警数、观点数、K 线数
- 查看图表标记、信号证据和最新观点

适用场景：

- 快速复核某个资产的当前信号
- 观察不同时间周期下的结构变化
- 对照观点摘要和图表证据

### 4.4 回放实验室

用途：提交历史回放任务，并查看最近的回放记录。

提交回放步骤：

1. 打开“回放实验室”
2. 选择“开始日期”和“结束日期”
3. 选择“配置版本”
4. 点击“提交回放任务”
5. 页面提示“回放任务已提交”后，历史列表会刷新

页面右侧会显示配置版本列表，方便确认当前回放关联了哪个版本。

### 4.5 运行历史

用途：查看历史分析快照和详细结果。

使用方式：

1. 打开“运行历史”
2. 在左侧“快照列表”中选择一条记录
3. 在右侧查看：
   - 运行 ID
   - 创建时间
   - 降级资产
   - 告警
   - BTC / ETH 的置信度和证据

适合用于复盘某次分析输出或排查异常告警。

## 5. 常见问题

### 5.1 登录后页面空白或数据不更新

优先检查以下进程是否都在运行：

- `pnpm dev`
- `pnpm worker`

### 5.2 点击“运行分析”或“提交回放任务”后没有结果

通常说明任务已写入队列，但 Worker 没有处理。请检查 `pnpm worker` 是否仍在运行。

### 5.3 健康检查怎么确认

访问 `GET /api/health`：

- 返回 `status: "ok"` 表示应用和数据库可读
- 返回 `503` 表示数据库不可用或运行时异常

## 6. 建议验证路径

如果要快速验证系统功能是否正常，建议按下面顺序操作：

1. 登录系统
2. 在“配置版本管理”创建一个新版本
3. 在“命令中心”点击“运行分析”
4. 在“分析工作台”切换资产和时间周期
5. 在“回放实验室”提交一个回放任务
6. 在“运行历史”查看最新快照
