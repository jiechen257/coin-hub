# Coin Hub Project Agent Rules

本文件补充当前项目的本地执行偏好。

## Commit 策略

- 涉及多个独立工作流的改动，默认按工作流分段提交 commit。
- 分段边界优先按这些维度切分：数据模型与迁移、后端接口、前端交互与样式、开发工具与文档。
- 准备 `commit` 或 `push` 前，若发现当前 diff 同时包含多个可独立回滚的改动面，应先拆分为多个 commit，再继续提交。
- 用户明确要求单个 commit 时，按用户要求执行。
- 若本次已经发生“应拆未拆”的情况，后续同类任务默认先做 commit 切分，再进入 push 或部署。

## 分支策略

- 默认直接在 `main` 分支开发。
- 只有用户明确要求独立功能分支、PR 流程、风险隔离或并行开发时，才创建非 `main` 分支。
